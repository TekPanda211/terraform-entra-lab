# v0.3.0-alpha - Identity Governance Engine

## Overview

This release introduces the BlackKnight One Identity Governance Engine.

The focus is governed access: how access is requested, approved, assigned, validated, reviewed, expired, and reported.

## Added

- Access Package framework
- Entitlement Management model
- Governance catalog
- Access Package standard
- Governance maturity model
- Administrative Unit planning
- Dynamic Group planning
- Access Review planning
- GDAP-inspired governance model
- Governance Engine PowerShell framework
- Standardized Governance Engine JSON output

## Principle

Access should never be assumed.

Access should be requested, approved, validated, reviewed, and backed by evidence.

## Next

Future releases will add Microsoft Graph discovery for catalogs, Access Packages, assignment policies, Access Reviews, and governance validation.
