\# Identity Requests Engine



\## Purpose



The Identity Requests Engine supports routine access requests and identity operations.



\## Supported Requests



\- Group membership request

\- Temporary access

\- PIM activation validation

\- Guest access request

\- Shared mailbox access

\- Distribution list access

\- Application access request

\- Access Package request



\## Request Pattern



Every identity request should include:



\- Requester

\- Target user

\- Requested access

\- Business justification

\- Approver

\- Expiration date when applicable

\- Validation result

\- Evidence report



\## Validation



Before completing a request, BlackKnight One should validate:



\- Least privilege alignment

\- Approval status

\- Existing access

\- Conflicting access

\- Expiration requirement

\- Audit evidence
